## 财富等级系统

财富等级根据用户的**现金 + 银行存款**总额确定，共分为10个阶段。

### 财富等级表

| 1 | 平民 | 0 | 25% | 3 |
| 2 | 小资 | 500 | 50% | 4 |
| 3 | 富豪 | 2,000 | 75% | 5 |
| 4 | 巨擘 | 5,000 | 100% | 6 |
| 5 | 权贵 | 15,000 | 125% | 7 |
| 6 | 领主 | 50,000 | 150% | 8 |
| 7 | 霸主 | 150,000 | 175% | 9 |
| 8 | 王者 | 500,000 | 200% | 10 |
| 9 | 传奇 | 1,500,000 | 250% | 15 |
| 10 | 神话 | 5,000,000 | 300% | 无限制(-1) |

### 财富等级判定逻辑

```python
def get_wealth_info(user_data):
    total = user_data["coins"] + user_data["bank"]  # 现金 + 银行存款
    for min_coin, name, rate in reversed(WEALTH_LEVELS):  # 从高到低遍历
        if total >= min_coin:
            return name, rate
    return "平民", 0.25
```

## 身价计算

身价是计算财富等级和排行榜的核心指标，包含三部分：**现金 + 银行存款 + 雇员潜在价值**。

### 公式

```
身价 = 现金 + 银行存款 + Σ(每个雇员的潜在价值)
```

### 雇员潜在价值计算

雇员的潜在价值 = `max(出售获得的钱, 赎身时雇主获得的钱)`

```python
# 出售获得的钱
sell_value = 雇员当前身价 × 出售返还率(默认0.8)

# 赎身时雇主获得的钱
# 赎身费用 = 购买记录中的价格（购买时实际支付的价格）
redeem_value = 赎身费用 × 赎身返还率(默认0.5)

# 雇员潜在价值
contractor_potential_value = max(sell_value, redeem_value)
```

### 配置参数

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `sell_return_rate` | 0.8 | 主动出售雇员时，可以收回其当前身价的80% |
| `redeem_return_rate` | 0.5 | 雇员赎身时，原雇主可以获得赎身费用的50% |

## 动态身价计算

用户的身价决定了被购买时的价格，是动态变化的。

### 公式

```
动态身价 = 基础身价 × (1 + 契约等级 × 每级契约身价加成)
```

### 计算步骤

1. **确定基础身价**：根据当前身价确定对应的财富等级基础身价
2. **获取契约等级**：用户被购买的次数
3. **计算身价加成**：每被购买一次，身价增加一定比例

### 代码实现

```python
async def calculate_dynamic_wealth_value(group_id, user_data, user_id):
    # 使用身价计算（包含雇员潜在价值）
    total = await calculate_wealth_value(group_id, user_data, user_id)

    # 根据身价确定基础身价
    base_value = 100  # 默认平民基础身价
    for min_coin, name, _ in reversed(WEALTH_LEVELS):
        if total >= min_coin:
            base_value = WEALTH_BASE_VALUES[name]
            break

    # 获取契约等级（被购买次数）
    contract_level = get_purchase_count(user_id)

    # 计算最终身价
    price_bonus = 0.15  # 每级契约身价加成，默认15%
    return base_value × (1 + contract_level × price_bonus)
```

### 配置参数

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `contract_level_price_bonus` | 0.15 | 每提升1级契约，身价增加15% |

## 签到收入计算

签到收入由多个部分组成，受多种因素影响。

### 收入组成

```
最终收益 = (基础收益 + 雇员加成 + 连续签到加成) × 受雇惩罚系数 + 银行利息
```

### 详细公式

```python
# 基础收益
base_income = 100  # 固定值
user_base_rate = 财富等级加成率  # 如平民25%，神话300%
base_with_bonus = base_income × (1 + user_base_rate)

# 雇员加成
contractor_dynamic_rates = 所有雇员的加成率之和
contract_bonus = base_with_bonus × contractor_dynamic_rates

# 连续签到加成
consecutive_bonus = 10 × (连续签到天数 - 1)

# 受雇惩罚（如果被雇佣）
if is_penalized:
    earned = (base_with_bonus + contract_bonus + consecutive_bonus) × 0.7
else:
    earned = base_with_bonus + contract_bonus + consecutive_bonus

# 银行利息
interest = 银行存款 × 0.01  # 1%日利率

# 最终收益
final_earned = earned + interest
```

### 明日预计收入

```python
total = base_with_bonus + contract_bonus + consecutive_bonus + interest

其中：
- base_with_bonus = 100 × (1 + 用户财富等级加成率)
- contract_bonus = base_with_bonus × 雇员总加成率
- consecutive_bonus = 10 × 连续签到天数  # 注意：明日预计使用当前连续天数
- interest = 银行存款 × 0.01
```

### 配置参数

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `employed_income_rate` | 0.7 | 被雇佣时签到收益只有正常的70% |

## 交易逻辑

### 购买逻辑

#### 基础价格

```
基础价格 = 目标动态身价

# 如果目标是管理员/群主
if target_role in ["owner", "admin"]:
    基础价格 ×= (1 + admin_price_bonus)  # 默认加成50%
```

#### 恶意收购（目标已有雇主）

```
总价格 = 基础价格 + 基础价格 × 恶意收购额外费用率
       = 基础价格 × (1 + takeover_fee_rate)

# 默认 takeover_fee_rate = 0.1，即额外支付10%
```

#### 交易流程

1. 检查购买者现金是否充足
2. 扣除购买者现金
3. 如果有原雇主：
   - 原雇主获得全部转让费（即总价格）
   - 移除原雇主与目标的雇佣关系
4. 建立新的雇佣关系
5. 增加目标的被购买次数

### 出售逻辑

```
出售获得 = 目标当前身价 × 出售返还率

# 默认 sell_return_rate = 0.8，即获得80%
```

#### 交易流程

1. 检查目标是否在购买者的雇员列表中
2. 计算出售价格
3. 增加购买者现金
4. 移除雇佣关系

### 赎身逻辑

```
赎身费用 = 购买记录中的价格

# 系统会查询 purchase_records 表获取该雇员被购买时的实际价格
# 如果没有购买记录（旧数据），则计算当前价格并记录到数据库

雇主获得补偿 = 赎身费用 × 赎身返还率

# 默认 redeem_return_rate = 0.5，即赎身费用的50%
```

#### 交易流程

1. 检查用户是否被雇佣
2. 查询 `purchase_records` 表获取购买价格
3. 如果没有购买记录，计算当前价格并记录到数据库（兼容旧数据）
4. 检查用户现金是否充足
5. 扣除用户现金（赎身费用）
6. 雇主获得补偿
7. 移除雇佣关系

#### 购买记录表

系统使用 `purchase_records` 表记录每次购买的历史：

| 字段 | 说明 |
|------|------|
| group_id | 群ID |
| owner_id | 雇主ID |
| contractor_id | 雇员ID |
| purchase_price | 实际支付价格（包含恶意收购费用、管理员加成等） |
| created_at | 购买时间 |

这样确保赎身费用与购买时的实际价格一致，避免身价波动导致的不公平。

### 配置参数

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `takeover_fee_rate` | 0.1 | 恶意收购额外费用率 |
| `sell_return_rate` | 0.8 | 主动出售返还率 |
| `redeem_return_rate` | 0.5 | 赎身时原雇主得款率 |
| `redeem_cost_rate` | 0.5 | 赎身费用比例 |
| `admin_price_bonus` | 0.5 | 管理员价格加成比例 |
| `owner_can_be_purchased` | false | 群主是否可被购买 |

## 雇员加成计算

雇员为雇主提供的签到收益加成由三部分组成。

### 公式

```
单个雇员加成率 = 基础加成率 + 契约等级加成 + 管理员加成 + 身价加成

总加成率 = Σ(所有雇员的加成率)
```

### 详细计算

```python
# 基础加成率 = 雇员财富等级加成率
base_rate = 雇员财富等级对应加成率  # 如平民25%，神话300%

# 契约等级加成
contract_level = 雇员被购买次数
rate_bonus_per_level = 0.075  # 默认每级7.5%
contract_bonus = contract_level × rate_bonus_per_level

# 管理员加成
if 雇员是群管理员:
    admin_bonus = 0.1  # 默认10%
else:
    admin_bonus = 0

# 身价加成（基于雇员总资产）
wealth_value_rate = 0.001  # 默认系数0.001
# 雇员总资产 = 现金 + 银行存款 + 雇员潜在价值（详见总资产计算章节）
contractor_total_assets = 雇员总资产
wealth_bonus = contractor_total_assets / 1000 × wealth_value_rate
# 例如：总资产10000的雇员提供 10000/1000×0.001 = 0.01 = 1% 加成

# 单个雇员总加成
contractor_rate = base_rate + contract_bonus + admin_bonus + wealth_bonus
```

### 示例

假设一个雇员：
- 财富等级：富豪（75%基础加成）
- 被购买次数：2次
- 是群管理员
- 总资产：50,000（现金+银行存款+雇员潜在价值）

计算：
```
基础加成 = 0.75
契约加成 = 2 × 0.075 = 0.15
管理员加成 = 0.1
身价加成 = 50000/1000 × 0.001 = 0.05

总加成率 = 0.75 + 0.15 + 0.1 + 0.05 = 1.05 (105%)
```

**注意**：身价加成现在基于雇员的总资产（包含其自身的雇员潜在价值），而非仅现金+银行存款。

### 配置参数

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `contract_level_rate_bonus` | 0.075 | 每级契约增益加成 |
| `admin_contractor_bonus` | 0.1 | 管理员雇员额外加成 |
| `wealth_value_bonus_rate` | 0.001 | 身价加成系数 |

## 兑换码系统

兑换码允许用户输入特定代码获得货币奖励。

### 兑换码属性

| 属性 | 说明 |
|------|------|
| code | 兑换码字符串 |
| description | 兑换码描述 |
| reward_amount | 奖励数量 |
| is_expired | 是否已过期（手动设置） |
| expire_time | 过期时间（格式：YYYY-MM-DD HH:MM） |
| max_uses | 最大使用次数（0表示无限制） |
| enabled_groups | 允许使用的群列表（空表示所有群） |

### 使用流程

1. 用户输入：`兑换 <兑换码>`
2. 系统检查：
   - 兑换码是否存在
   - 是否已过期（手动或时间）
   - 使用次数是否已达上限
   - 当前群是否允许使用
   - 用户是否已兑换过
3. 如果通过所有检查：
   - 发放奖励到用户现金
   - 增加兑换码使用次数
   - 记录用户兑换记录

### 兑换限制

- 每个用户每个兑换码只能使用一次
- 达到最大使用次数后，其他用户无法再使用
- 可以设置特定群才能使用某些兑换码

## AT机器人奖励系统

用户AT机器人时，有概率获得随机金币奖励。

### 奖励规则

```python
# 概率判定
probability = 0.3  # 默认30%概率
random_value = random(1, 1000)
if random_value <= probability × 1000:
    触发奖励

# 奖励金额
reward = random(reward_min, reward_max)  # 默认1-10金币
```

### 限制条件

| 限制 | 默认值 | 说明 |
|------|--------|------|
| 每日上限次数 | 5次 | 每个用户每天最多获得奖励的次数 |
| 时区 | Asia/Shanghai | 用于每日重置计数 |

### 奖励记录

系统会记录：
- 用户每次获得奖励的日期
- 每次奖励的金额
- 每日奖励次数和总金额

## 银行存取款系统

用户可以将现金存入银行获得利息，或从银行取款。

### 存款

```
存款后：
- 现金 -= 存款金额
- 银行存款 += 存款金额
```

### 取款

```
取款后：
- 银行存款 -= 取款金额
- 现金 += 取款金额
```

### 银行利息

```
每日利息 = 银行存款 × 0.01  # 1%日利率

# 利息在签到时自动发放到银行存款
```

### 利息计算示例

| 银行存款 | 每日利息 |
|----------|----------|
| 1,000 | 10 |
| 10,000 | 100 |
| 100,000 | 1,000 |
